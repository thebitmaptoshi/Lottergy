# .npmrc

**What:** pnpm tuning so Expo + Metro bundler work cleanly in a pnpm monorepo.

**Why:** Default pnpm `node-linker=isolated` creates symlinked `node_modules` trees that Metro/Expo can't always resolve (especially React Native peer deps). The Expo team recommends `node-linker=hoisted` for RN monorepos.

## Settings

- `node-linker=hoisted` — flat `node_modules` like npm/yarn. Metro is happy.
- `strict-peer-dependencies=false` — RN's loose peer-dep landscape would otherwise block installs.
- `auto-install-peers=true` — silently install peer deps when a dep needs them (saves a manual `pnpm add`).

## Key invariants

- Do NOT switch back to `isolated` linker unless every Expo + RN issue is independently verified resolved.
- This file is committed (not local-only) — shared across machines.
