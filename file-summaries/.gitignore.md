# .gitignore

**What:** Excludes generated, environment, OS, and IDE artifacts from version control.
**Why:** Locked Phase 0 list in GOAL.md, extended with Python ignores so `powerball-legacy/` doesn't leak `__pycache__/` or virtualenvs.

## Sections

- **Node / JS (forward-looking):** `node_modules/`, `.expo/`, `dist/`, `build/`, `coverage/`, `.vitest/` — keep the upcoming Expo monorepo clean from day one.
- **Python (for `powerball-legacy/`):** `__pycache__/`, `*.pyc`, `*.pyo`, `.venv/`, `venv/`.
- **Logs / env / OS:** `*.log`, `.env`, `.env.*`, `.DS_Store`, `Thumbs.db`.
- **IDE:** `.idea/` only. `.vscode/` is intentionally not ignored so shared workspace settings can be committed if useful.
- **Claude Code:** `.claude/settings.local.json` only. If a shared `.claude/settings.json` is added later it will be committed.

## Key invariants

- Never commit secrets — `.env` and `.env.*` cover the common cases; pre-flight any new env-loading lib for additional patterns.
- Keep the file short and grouped. New entries belong under the relevant section header.
- The `top *.txt` legacy run outputs in `powerball-legacy/` are intentionally committed as reference baselines — do not add a glob that hides them.
